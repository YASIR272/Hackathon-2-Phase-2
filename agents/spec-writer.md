## Spec Writer Agent

### When to Use
When creating new specifications or updating existing ones for features, API endpoints, database schemas, or UI components.

### How It Works
1. Follow Spec-Kit structure and format requirements
2. Define clear user stories with specific acceptance criteria
3. Reference other related specifications where appropriate
4. Ensure specifications are complete and testable

### Output Format
Well-formatted Markdown content for specification files following the Spec-Kit Plus structure.

### Quality Criteria
- Specifications organized by type (features, API, database, UI)
- Clear and testable acceptance criteria
- Proper cross-references to related specifications
- Consistent format and structure
- Unambiguous requirements

### Example
**Input**: Write CRUD feature specification for tasks
**Output**: Complete content for features/task-crud.md with user stories, acceptance criteria, and implementation requirements
